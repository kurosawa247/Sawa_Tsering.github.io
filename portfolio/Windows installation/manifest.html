Evidence title: Windows installation

Evidence description:

After installing Windows on a system, I performed a structured post-installation validation. The goal was to verify that the operating system, drivers, settings, and applications were correctly installed and functioning properly before the system is delivered to a user.

This is an important preventive troubleshooting step to avoid future incidents.

What I checked

  • Windows installation completeness

  • Device Manager (driver errors / missing drivers)

  • Network connectivity

  • Windows Update status

  • Security settings

  • User accounts & permissions

  • Required applications installed

  • System settings configuration

  • Activation status

  • Firewall & Defender status

  • Disk configuration

  • System performance basics

Troubleshooting mindset applied

Before solving problems, I learned that it is essential to validate configuration first. Many issues are caused by:

  • wrong settings

  • missing drivers

  • incomplete updates

  • misconfigured network

  • missing software dependencies

By checking these items first, problems can be prevented or solved faster.

I applied a checklist approach instead of random testing.
