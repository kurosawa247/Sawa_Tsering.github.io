Evidence title: WPL: Backing up/restoring

Evidence description:

I helped test, verify, and prepare backups for the Proxmox environment and the CERcuits servers. I also tried out different backup scenarios by restoring backups in a new Proxmox environment, which allowed me to check whether the data and virtual machines were restored correctly. In this way, I could confirm that the backups were actually usable in a real recovery situation.

In addition, I worked on the backup server itself, where we noticed that the restore process was too slow. To solve this issue, we replaced the server’s CPU and RAM so that backup and restore processes could run more efficiently. This gave me insight into how hardware performance can directly affect the reliability and speed of backup procedures.

There were also virtual machines that were actively used by colleagues, so we had to schedule the backups at appropriate times without interrupting their work. This taught me that backups must not only be technically correct, but also carefully planned to fit the organization’s daily operations.

What this shows

  • Testing and restoring backups.

  • Verifying whether backups are usable in practice.

  • Improving backup server performance.

  • Planning backups without disrupting users.

  • Understanding the importance of backup reliability in critical systems.
